# linux常用命令：head命令详解

在 Linux 日常使用和运维中，`head` 是查看文件开头内容的常用命令。无论是快速检查配置文件前几行、预览日志开头、确认数据文件格式，还是配合管道截取命令输出，`head` 都很实用。

---

## 一、`head` 是什么？

`head` 用于显示文件或命令输出的前几行内容。

它主要用于：

- 查看文件开头内容
- 预览日志、配置文件、数据文件
- 截取命令输出前几行
- 配合管道做快速筛选

说明：

- `head` 默认显示前 10 行
- 如果要看文件末尾内容，通常使用 `tail`

---

## 二、常见使用语法与重点参数

```bash
head [选项] [文件]
```

重点参数说明

- `-n <行数>`：显示前多少行
- `-c <字节数>`：显示前多少字节
- `-q`：处理多个文件时不显示文件名头部
- `-v`：处理多个文件时始终显示文件名头部

---

## 三、常用命令与使用场景

### 1. 查看文件前 10 行

```bash
head app.log
```

这是最基础的用法，适合快速预览文件开头内容。

### 2. 查看前 20 行

```bash
head -n 20 app.log
```

当默认 10 行不够时，通常会指定更明确的行数。

### 3. 查看前 100 字节

```bash
head -c 100 file.txt
```

适合只想快速看文件开头一小段内容时使用。

### 4. 同时查看多个文件开头

```bash
head file1.txt file2.txt
```

会依次输出多个文件的前 10 行，并带上文件名头部。

### 5. 截取命令输出前几行

```bash
ps -ef | head
```

适合在输出很长时先快速看前几行结果。

### 6. 查看配置文件开头注释或说明

```bash
head -n 30 /etc/nginx/nginx.conf
```

很多配置文件开头会有注释、示例和全局配置，这种场景下 `head` 很方便。

### 7. 预览 CSV 或数据文件格式

```bash
head -n 5 data.csv
```

适合先确认表头、字段顺序和分隔格式。

## 四、常见问题与建议

### 1. `head` 和 `tail` 的区别是什么？

- `head`：看开头
- `tail`：看结尾

如果你想快速看文件最前面的内容，用 `head`；如果想看最新日志或文件末尾内容，用 `tail`。

### 2. `head` 适合查看大文件吗？

适合。因为它只读取前面一小部分内容，比直接输出整个文件高效得多。

### 3. `head -n` 和 `sed -n '1,10p'` 有什么区别？

两者都能取前几行，但 `head -n` 更直接，适合简单场景；`sed` 更适合复杂范围提取。

### 4. 为什么有时会看到文件名前缀？

当一次查看多个文件时，`head` 默认会显示类似下面的头部，便于区分来源：

```bash
==> file1.txt <==
```

如果不想显示，可以加 `-q`。

### 5. 实际工作里最常用的是哪几个组合？

- `head file`
- `head -n 20 file`
- `head -c 100 file`
- `命令 | head`

## 五、常用命令速查

```bash
head file.txt
head -n 20 file.txt
head -c 100 file.txt
head file1.txt file2.txt
head -q file1.txt file2.txt
ps -ef | head
cat app.log | head -n 30
```

## 六、总结

`head` 是 Linux 下查看文件开头内容的基础命令。建议优先熟练掌握以下几条：

```bash
head file.txt
head -n 20 file.txt
head -c 100 file.txt
head file1.txt file2.txt
命令 | head
```

掌握这些用法后，大多数“预览文件开头、查看表头、确认配置格式、截取输出前几行”的场景都能快速处理。
