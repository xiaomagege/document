# top命令详解

## 1. 命令简介
`top` 是 Linux 下实时查看系统资源和进程状态的命令。它会周期性刷新，展示 CPU、内存、负载和进程列表。

## 2. 基本用法
```bash
top
```

常用启动参数：
- `top -c`：显示完整命令行
- `top -d 2`：每 2 秒刷新一次
- `top -n 5`：刷新 5 次后退出
- `top -p 1234`：只看 PID 为 1234 的进程
- `top -u root`：只看指定用户进程
- `top -b -n 1`：批处理模式输出一次（适合脚本）

## 3. 界面结构

### 3.1 第一行：系统时间与负载
典型内容包含：
- 当前时间
- 系统运行时长
- 登录用户数
- `load average`（1/5/15 分钟负载）

关注建议：
- `load average(1m)` 持续大于 CPU 核数：需要关注系统是否过载。
- `1m > 5m > 15m`：负载在持续上升，需要尽快定位热点进程。
- `1m < 5m < 15m`：负载在回落，通常是瞬时高峰后的恢复期。

### 3.2 第二行：Tasks 进程状态
常见状态：
- `R`：运行中
- `S`：休眠
- `D`：不可中断等待
- `Z`：僵尸

关注建议：
- 出现持续的 `D` 状态进程：优先关注 IO/存储链路问题。
- `Z` 僵尸进程持续累积：需要关注父进程回收逻辑。

### 3.3 第三行：CPU 使用率
主要字段：
- `us`：用户态 CPU
- `sy`：内核态 CPU
- `ni`：调整优先级进程占用
- `id`：空闲 CPU
- `wa`：IO 等待
- `hi`：硬中断
- `si`：软中断
- `st`：虚拟机被“偷走”的 CPU

关注建议：
- `wa > 10%`：需要重点关注 IO 瓶颈（磁盘/网络存储）。
- `id < 20%` 且持续：CPU 空闲不足，需要关注 CPU 饱和。
- `sy` 持续偏高（如 >20%）：需要关注系统调用、内核开销或 IO 压力。
- `si/hi` 持续偏高（如 >10%）：需要关注中断风暴、网卡或驱动问题。
- `st > 5%`：虚拟化环境可能存在宿主机 CPU 争抢。

### 3.4 第四/五行：内存与 Swap
- `Mem`：物理内存
- `Swap`：交换分区

常见列：
- `total`：总量
- `free`：空闲
- `used`：已用
- `buff/cache`：缓存（可回收）

关注建议：
- `Swap used` 持续增长：需要关注内存不足或内存回收压力。
- `free` 很低但 `buff/cache` 很高：通常不算异常，先看是否发生 swap。
- 业务变慢且 swap 活跃：优先检查大内存进程与内存泄漏风险。

### 3.5 进程列表
常见字段：
- `PID`：进程 ID
- `USER`：所属用户
- `PR`/`NI`：优先级
- `VIRT`：虚拟内存
- `RES`：物理内存驻留
- `SHR`：共享内存
- `S`：进程状态
- `%CPU`：CPU 占比
- `%MEM`：内存占比
- `TIME+`：累计 CPU 时间
- `COMMAND`：命令名/命令行

关注建议：
- 单进程 `%CPU` 长时间接近 100%（单核）或更高（多线程）：说明该进程是 CPU 热点。
- `%MEM` 持续上升且不回落：需要关注内存泄漏或缓存失控。
- `S` 列长期为 `D`：说明进程长期卡在不可中断等待。

## 3.6 参数关注速查表
| 参数 | 需要关注的情况 | 可能方向 |
| --- | --- | --- |
| `load average(1m)` | 持续 > CPU 核数 | 系统过载 |
| `wa` | `> 10%` | IO 瓶颈 |
| `id` | `< 20%` 且持续 | CPU 紧张 |
| `sy` | 持续偏高（如 `>20%`） | 内核/系统调用/IO 开销高 |
| `st` | `> 5%` | 虚拟机 CPU 被争抢 |
| `D` 状态进程 | 持续出现 | IO/存储等待 |
| `Swap used` | 持续增长 | 内存压力或泄漏风险 |
| `Z` 状态进程 | 持续累积 | 进程回收异常 |

## 4. 运行中的常用交互按键
- `h`：帮助
- `q`：退出
- `c`：切换命令名/完整命令行
- `P`：按 `%CPU` 排序
- `M`：按 `%MEM` 排序
- `T`：按 `TIME+` 排序
- `1`：展开每个 CPU 核心使用率
- `k`：结束进程（输入 PID）
- `r`：修改 nice 值（调整优先级）
- `u`：过滤指定用户
- `f`：选择显示字段
- `o`：设置排序/过滤条件

## 5. 实用示例

查看某 Java 进程：
```bash
top -p 12345 -c
```

每秒刷新并保留 10 次快照：
```bash
top -b -d 1 -n 10 > top.log
```

仅查看 `nginx` 用户进程：
```bash
top -u nginx
```

## 6. 使用建议
- 日常使用优先 `top -c`，便于直接看到完整命令行。
- 需要给脚本或监控系统采样时，用 `-b -n` 输出固定次数。
- 结合 `P`、`M`、`T` 排序键，可以快速定位高 CPU/高内存/长时间运行进程。
